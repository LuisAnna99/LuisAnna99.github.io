SQL基础知识

SQL简介

SQL（Structured Query Language，结构化查询语言）是一种用于管理关系数据库的语言，通过一系列的语句和命令来执行数据定义、数据查询、数据操作和数据控制等功能,包括数据插入、查询、更新和删除，数据库模式创建和修改，以及数据访问控制。

SQL工作原理

  • SQL是一种声明性语言，它不像程序语言那样需要编译和链接，而是在运行时解释执行。

  • SQL的工作原理是一个声明->解析->优化->执行的自动化过程。

    • 查询优化器 (The Optimizer) 的工作原理：

    • 执行引擎与存储引擎 (Execution & Storage) 的工作原理：
    • 执行引擎就像一个项目经理，它不直接干活，而是指挥存储引擎这个“工人团队”去完成具体任务
      • 缓冲池 (Buffer Pool)： 数据库的主内存区域，缓存频繁访问的数据页。这是性能优化的核心，极大减少了直接读磁盘的次数。
      • 重做日志 (Redo Log)： 一种“写前日志”。任何数据修改之前，都会先把这个动作记录到Redo Log。即使数据库突然崩溃，重启后也能根据Redo Log重新执行这些操作，保证数据不丢失（持久性）。
      • 磁盘数据文件： 数据最终持久化存储的地方。

  • SQL强大之处在于，用户无需关心底层复杂的数据存储和检索细节，只需专注于业务逻辑，数据库会自动且智能地以尽可能高效的方式完成工作。

SQL语言结构

SQL语言由以下几个部分组成：

  • 数据定义语言（Data Definition Language，DDL）：用于定义数据库对象，如数据库、表、视图、索引等。
  • 数据操作语言（Data Manipulation Language，DML）：用于操作数据库对象，如插入、删除、更新和查询数据。
  • 数据控制语言（Data Control Language，DCL）：用于控制数据库对用户的访问权限和事务处理。
  • 数据查询语言（Data Query Language，DQL）：用于查询数据库中的数据。

SQL语句分类

SQL语句分为以下几类：

  • 数据定义语句：用于定义数据库对象，如创建、删除、修改数据库、表、视图、索引等。

  • 数据操作语句：用于操作数据库对象，如插入、删除、更新和查询数据。

  • 数据控制语句：用于控制数据库对用户的访问权限和事务处理。

  • 数据查询语句：用于查询数据库中的数据。

  • 事务控制语句：用于管理事务，如开始事务、提交事务、回滚事务等。

  • 系统管理语句：用于管理数据库，如备份和恢复数据库、设置数据库参数等。

  • 工具语句：用于执行诸如调试和性能分析等任务。

    SQL语法规则

    SQL语法规则是指SQL语言的语法规则，它规定了SQL语言的基本语法结构、关键字、保留字、数据类型、函数、运算符、表达式、注释等。

    基本语法结构

    SQL语言的基本语法结构包括：

    • 大小写不敏感：SELECT 与 select 是相同的。
    • 语句分隔符：一条SQL语句必须以分号（;）结尾。
    • 注释：SQL语言支持单行注释和多行注释，单行注释以“–”开头，多行注释以“/”和“/”开头和结尾。
    • 一些最重要的SQL命令：
    • SELECT：从数据库表中提取数据。
    • INSERT INTO：向数据库表中插入数据。
    • UPDATE：更新数据库表中的数据。
    • DELETE FROM：从数据库表中删除数据。
    • CREATE DATABASE：创建数据库。
    • CREATE TABLE：创建数据库表。
    • CREATE：创建数据库对象，如数据库、表、视图、索引等。
    • ALTER：修改数据库对象，如表结构、索引等。
    • DROP：删除数据库对象。
    • GRANT：授予用户权限。
    • REVOKE：回收用户权限。
    • COMMIT：提交事务。
    • ROLLBACK：回滚事务。
    • SET：设置数据库参数。
    • USE：选择当前数据库。

关键字

SQL语言的关键字是指SQL语言中具有特殊含义的单词，如SELECT、FROM、WHERE、AND、OR、ORDER BY等

保留字

SQL语言的保留字是指SQL语言中具有特殊用途的单词，如UNION、JOIN、GROUP BY、HAVING、LEFT OUTER JOIN等。

  • 一些常用的关键字和保留字

    • where: 用于指定查询条件

      *condition:筛选条件

    • order by: 用于对结果集进行排序

      *column_name:排序依据
      *ASC|DESC:升序(默认)或降序

    • group by: 用于对结果集进行分组

      *column_name:分组依据
      *AGGREGATE_FUNCTION:聚合函数，如SUM、AVG、MAX、MIN等

    • limit: 用于限制返回结果的数量

      *number:返回结果的数量

    • join: 用于连接多个表

      *table_name1:第一个表名
      *table_name2:第二个表名
      *column_name:连接条件
      *jion:连接类型，可以是INNER JOIN、LEFT OUTER JOIN、RIGHT OUTER JOIN或FULL OUTER JOIN

    • union: 用于合并两个或多个SELECT语句的结果集

    *table_name1:第一个表名
    *table_name2:第二个表名
    *column_name:连接条件

    • having: 用于对分组后的结果集进行筛选

    *column_name:分组依据
    *AGGREGATE_FUNCTION:聚合函数，如SUM、AVG、MAX、MIN等
    *condition:筛选条件

    • DISTINCT: 用于返回唯一不同的值/去除重复值

数据类型

SQL语言支持的数据类型包括：

  • 整型：SMALLINT、INTEGER、BIGINT
  • 浮点型：REAL、DOUBLE、DECIMAL
  • 字符串型：CHAR、VARCHAR、TEXT
  • 日期型：DATE、TIME、TIMESTAMP
  • 布尔型：BOOLEAN
  • 二进制型：BINARY、VARBINARY、IMAGE
